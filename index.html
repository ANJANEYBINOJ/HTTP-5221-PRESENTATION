Live Football Scoreboard
Usability Test Results

Project: Live Football Scoreboard Web Application
Tested By: Anjaney Binoj
Test Date: Week of 08-05-2025
Participants: 4-6 users (Football coaches, sports enthusiasts, and general users)

Executive Summary

The Live Football Scoreboard usability test evaluated core match management functions including score tracking, timer controls, foul management, data saving, and interface customization. The test identified several critical usability issues that impact user task completion and overall satisfaction with the application.

Key Finding: While users can complete basic scoring tasks, significant issues exist with timer controls, data export functionality, and overall interface clarity that require immediate attention.

Test Overview

Participants

  • Football Coaches & Enthusiasts: 2-3 participants with match management experience
  • General Users: 2-3 participants with basic technical skills and football knowledge

Testing Environment

  • Platform: Web application at anjaneybinoj.github.io/Pet_project/
  • Duration: 10-15 minutes per session
  • Method: Task-based usability testing with think-aloud protocol

Test Scenarios

Task 1: Basic Match Management
Start a match, add 2 goals to Team A and 1 goal to Team B
Task 2: Timer and Foul Management
Start match timer, add 2 fouls to Team A, pause timer
Task 3: Match Completion and Export
Save match result and export data as CSV
Task 4: Interface Customization
Toggle dark mode and reset match

Key Results

75% Overall Task Success Rate
12 min Average Session Time
6 Critical Issues Found
3.2/5 User Satisfaction Score

Critical Issues Identified

High Priority: Timer Control Confusion

Problem: Users struggled to understand the difference between "Start Match" and timer controls.
Impact: 4 out of 6 participants failed to properly start the timer, leading to task abandonment.
Evidence: Users clicked "Start Match" expecting the timer to begin automatically.

High Priority: Data Export Functionality

Problem: CSV export feature was not discoverable and didn't work as expected.
Impact: 5 out of 6 participants couldn't locate or successfully use the export function.
Evidence: Users looked for export options in menus or settings that don't exist.

Medium Priority: Foul Counter Visibility

Problem: Foul counters are not clearly labeled or visually prominent.
Impact: 3 out of 6 participants didn't notice foul tracking functionality initially.
Evidence: Users spent time looking for foul management features.

Medium Priority: Save Match Result Feedback

Problem: No confirmation message when saving match results.
Impact: Users uncertain whether their data was successfully saved.
Evidence: Multiple participants clicked save button repeatedly.

User Feedback

Positive Comments

  • "The scoring buttons are easy to use"
  • "Clean, simple interface design"
  • "Dark mode is a nice touch"
  • "Team names are clear and visible"

Areas of Concern

  • "I don't understand how to start timing"
  • "Where do I export the data?"
  • "Is the match actually saved?"
  • "The foul buttons are confusing"

Recommendations

Immediate: Improve Timer Controls

Solution: Combine "Start Match" with timer functionality or provide clear labels distinguishing match state from timer controls. Add visual indicators showing when timer is active.

Immediate: Fix Export Functionality

Solution: Make export button more prominent, ensure it works reliably, and provide feedback when export completes. Consider adding export options to a more discoverable location.

Short-term: Enhance Visual Feedback

Solution: Add confirmation messages for save actions, improve foul counter labels, and provide status indicators for match state and timer status.

Long-term: User Experience Improvements

Solution: Conduct user flow analysis to optimize task completion paths, add help documentation or tooltips, and implement user onboarding for first-time users.
